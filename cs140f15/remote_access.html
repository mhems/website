Guide to remotely accessing LNG computers

Note:

Since you are remotely accessing the system from the command line, you will only be able to use the command line and as such won't be able to open a gedit window or run other standalone applications like Eclipse. Thus, the command gedit myFile.txt & or even gedit myFile.txt will give an error since an application window can't open from PuTTY and gedit has no equivalent command-line program. This is a motivating factor for using in-terminal editing programs like Vim, Emacs, and Nano, which would allow you to edit from inside the terminal. If you would like to edit your files using an editor or program of your choice, you can ftp the files to your local computer, edit them, and then ftp the files back to the lab computers. To get an ftp client, download PSFTP along with PuTTY. If on Unix/Linux/Mac, you can simply use the ftp or scp commands in the terminal.

Unix/Linux/Mac Users:

  1. Open up a terminal
  2. Run the command ssh username@bingsuns2.cc.binghamton.edu
  3. Enter your PODS password and press Enter
  4. You should now be able to ls and see your lab file directory
  5. When you are done, run exit to end your session and return to your computer

PC Users:

  1. First time only: download PuTTY
    • For just the PuTTY application itself (all you really need), download just putty.exe and store it in a safe place with a shortcut on your Desktop
    • You can alternatively download putty-0.xx-installer.exe which will download the PuTTY suite
  2. Launch PuTTY
  3. Type username@bingsuns2.cc.binghamton.edu into the Host Name text box
  4. Ensure SSH is checked as the Connection type
  5. Select the Open button at the bottom right to launch your session
  6. Enter your PODS password and press Enter
  7. You should now be able to ls and see your lab file directory
  8. When you are done, run exit to end your session and close PuTTY
