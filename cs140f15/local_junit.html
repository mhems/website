Using JUnit without Eclipse

JUnit is really just a bunch of third party code some people wrote. Eclipse gives us a nice graphical view of JUnit, but its still using the same code under the hood.

So the good news is, we can still get the benefits of JUnit outside of Eclipse (i.e. on the command line)

Download this simple script to configure your linux environment to be able to run JUnit.

Note that this was written and tested on a linux environment. It should work on Mac or git-bash on Windows.

  1. Download the script (called install_junit.sh)
    • It doesn't matter where you download or run the script
  2. Make the script runnable with chmod u+x install_junit.sh
  3. Invoke the script with ./install_junit.sh
    • Once the script is run, you won't need the script again
  4. Update these changes with source ~/.bashrc
After the above steps, all should be set up with minor changes to your build process
  • You still compile with javac -d . *.java or whichever files you want to (re)compile
  • To run a JUnit class, run junit PKG_NAME.CLASS_NAME
    • e.g. junit assignment06.TestGPA
  • To run a normal class with a main method, its the same as before, java -cp . PKG_NAME.CLASS_NAME
