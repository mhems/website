Git Basics to submit your code

Assuming you haven't pushed or cloned (starting from scratch)

Create a new git repository: git init

This should not trigger any errors

Add the files you want to submit: git add FILE1 FILE2 ...

This should not trigger any errors

Alternatively, you can run git add . which will add every file in the current directory

Or you can run, git add *.java which will add every file that ends with the java extension

Note: It is not necessary for you to submit .class files or .java~ files. The former is the executable byte code that we can obtain from compiling your source code. The latter are backup files that editors make for your convenience. Do not submit your files in an archive such as tar, rar, or zip.

You can confirm the files you wanted to add are indeed added by running git status. If you were successful in adding them, the output should list those files as "Changes ready to be committed"

Commit your added files: git commit -m "Ready to submit"

The message between the double quotes should be a message describing what your commit entails

This may trigger an error that starts with "Tell me who you are". To rectify this, run the two commands:

git config --global user.name "Your Name"

git config --global user.email "your bing email"

and re-run the commit command. You should only have to do this once per machine.

If all succeeded, your output should say "X file(s) changed, Y insertions, Z deletions" and then a list of items that look like "create mode NNN file"

Sync your commit to gitlab:

git push https://gitlab.com/bu-cs140-admins/PROJECT_NAME-YOUR_PODS_USERID.git master

where PROJECT_NAME is the name of the project (e.g. assignment02, lab02, exampractice) and YOUR_PODS_USERID is your PODS user name

Note you will need to enter your gitlab user id and your gitlab password

This is the crucial step that needs to succeed for your submission to be valid.

If all went well, you should see a message whose last line ends in "master -> master"

You can confirm this by pointing your browser to https://gitlab.com/bu-cs140-admins/PROJECT_NAME-YOUR_PODS_USERID and navigating to Files and checking the files you want to upload are there with an appropriate timestamp.

There are several errors you could get with this command, the most likely saying something like "fatal ... can't find refspec ..."

Check that the url is properly formed and re-enter your password carefully.

If this does not correct the error, post to gitlab with the command you're trying to execute and the full error message you get

Assuming you have already pushed at least once and want to upload changes to the same place

We can skip git init since its already initialized

Start with the git add step outlined above to add the files you've changed.

Then git commit ... and git push ... as outlined above

Assuming you would like to grab code from gitlab in a fresh directory

Clone the repository to your local machine:

git clone https://gitlab.com/bu-cs140-admins/PROJECT_NAME-YOUR_PODS_USERID.git

You now have your files to examine, edit, and re-submit if you choose, using the commands outlined above
