Git Basics to submit your code

Starting from scratch (you haven't pushed or cloned)

  1. Create a new git repository: git init

    2. This should not trigger any errors

  2. Add (tell git about) the files you want to submit: git add FILE1 FILE2 ...

    3. This should not trigger any errors

    You can add all files in the current directory with git add .

    You can add all java files in the current directory with git add *.java

    Note: It is not necessary for you to submit .class files or .java~ files. The former is the executable byte code that we can obtain from compiling your source code. The latter are backup files that editors make for your convenience. Do not submit your files in an archive such as tar, rar, or zip.

  3. Commit your changes locally: git commit -m "Ready to submit"

    4. The message between the double quotes is for your use - it should describe what you have last changed. Since this is our first commit, "Ready to submit" or "initial commit" is fine.

    This may trigger an error that starts with "Tell me who you are". To rectify this, run the two commands:

    git config --global user.name "Your Name"

    git config --global user.email "your bing email"

    and re-run the commit command. You should only have to do this once per machine.

    If all succeeded, your output should say "X file(s) changed, Y insertions, Z deletions" and then a list of items that look like "create mode NNN file"

  4. Sync your local changes to your gitlab account: git push https://gitlab.com/bu-cs140s16-admins/PROJECT_NAME-YOUR_PODS_USERID.git master

    5. where PROJECT_NAME is the name of the project (e.g. assignment02, lab02, exampractice) and YOUR_PODS_USERID is your PODS user name

    Note you will need to enter your gitlab user id and your gitlab password

    The shell will NOT show your password being typed, just type it and press ENTER

    This is the crucial step that needs to succeed for your submission to be valid.

    If all went well, you should see a message whose last line ends in "master -> master"

    You can confirm this by pointing your browser to https://gitlab.com/bu-cs140s16-admins/PROJECT_NAME-YOUR_PODS_USERID and navigating to Files and checking the files you wanted to upload are there with an appropriate timestamp.

    There are several errors you could get with this command

    • fatal ... can't find refspec...

      • Check that the url is properly formed and re-enter your password carefully.

    • Updates were rejected because the remote contains work that you do not have locally

      • Check out this page

    • If your error is neither of these two, post to gitlab with the command you're trying to execute and the full error message you get
