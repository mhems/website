Common Git commands used in this course

Note: Command line usage information typically follows the convention that optional arguments are enclosed in square brackets ( [] ). For instance, you can either run git status to ask git about all your files or run git status file1.txt to ask git about the status of file1.txt



To ask git for help: git help [command]

To ask git about the status of your files: git status [files]

To see a history of your commits: git log

To clone a new remote repository to your local computer: git clone URL

To create a new repository: git init

To add any files you may have changed to be committed: git add file1 [file2...]

To remove a commited file you no longer want: git rm file1 [file2...]

To commit changes: git commit -m ' message'

To pull any changes made to the repository to your local computer: git pull URL master

To push your local commited changes to gitlab: git push URL master
